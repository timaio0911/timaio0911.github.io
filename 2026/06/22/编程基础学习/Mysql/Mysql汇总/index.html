MySQL 基础与进阶

Part1 – DQL 数据查询

1. SELECT 基础查询

指定要查询的列，逗号分隔

SELECT student_name, major, email
FROM students;

2. WHERE 条件过滤

逻辑运算符

运算符 说明
AND 逻辑与，所有条件都为真才为真
OR 逻辑或，至少一个条件为真就为真
NOT 逻辑非，取反 
-- 查询计算机科学与技术专业的男学生
SELECT * FROM students WHERE major = '计算机科学与技术' AND gender = '男';

-- 查询成绩在80分以上或选课日期在2023-10-01之后的选课记录
SELECT * FROM enrollments WHERE grade > 80 OR enrollment_date > '2023-10-01';

-- 查询不是软件工程专业的学生
SELECT * FROM students WHERE NOT major = '软件工程';

模糊查询 LIKE

[!info]

  • %：匹配任意长度的任意字符（0个或多个）
  • _：匹配单个任意字符
-- 查询姓张的学生
SELECT * FROM students WHERE student_name LIKE '张%';

-- 查询名字第二个字是'三'的学生
SELECT * FROM students WHERE student_name LIKE '_三%';

-- 查询邮箱以'.com'结尾的学生
SELECT * FROM students WHERE email LIKE '%.com';

-- 查询课程名称不包含'数据'的课程
SELECT * FROM courses WHERE course_name NOT LIKE '%数据%';

3. 排序与限制

ORDER BY 排序

  • ASC：升序（默认）
  • DESC：降序
  • 可以指定多个列进行排序

LIMIT 限制

用于限制查询结果返回的行数

  • offset：可选参数，表示从第几行开始返回（起始行是0）
  • row_count：要返回的行数

4. 聚合函数

GROUP BY 分组查询

GROUP BY 子句用于将查询结果按照一个或多个列进行分组，然后对每个组应用聚合函数。

HAVING 分组后过滤

HAVING 子句用于在 GROUP BY 分组之后，对聚合结果进行过滤。WHERE 子句不能用于过滤聚合函数的结果。

Part2 – 日期函数

常用日期函数

函数 说明
NOW() 获取当前日期时间
CURDATE() 获取当前日期
CURTIME() 获取当前时间
DATE() 提取日期部分
TIME() 提取时间部分
YEAR() 提取年份
MONTH() 提取月份
DAY() 提取日期
HOUR() 提取小时
MINUTE() 提取分钟
SECOND() 提取秒

DATE_FORMAT() 日期格式化

SELECT DATE_FORMAT(NOW(), '%Y-%m-%d %H:%i:%s');
-- 结果: 2025-06-11 14:30:25

SELECT DATE_FORMAT('2025-06-11', '%Y年%m月%d日');
-- 结果: 2025年06月11日

SELECT DATE_FORMAT('2025-06-11 14:30:25', '%W, %M %e, %Y');
-- 结果: Wednesday, June 11, 2025

[!tip]
常用格式化符号：

  • %Y：四位年份
  • %m：月份（01-12）
  • %d：日期（01-31）
  • %H：小时（00-23）
  • %i：分钟（00-59）
  • %s：秒（00-59）
  • %W：星期名称

日期提取函数

-- 提取日期部分
SELECT DATE('2025-06-11 14:30:25');  -- 2025-06-11

-- 提取时间部分
SELECT TIME('2025-06-11 14:30:25');  -- 14:30:25

-- 提取年份
SELECT YEAR('2025-06-11');  -- 2025

-- 提取月份
SELECT MONTH('2025-06-11');  -- 6

-- 提取日期
SELECT DAY('2025-06-11');  -- 11

-- 返回季度
SELECT QUARTER('2025-06-11');  -- 2

-- 返回周数
SELECT WEEK('2026-04-28');  -- 默认 mode=0 周日起始
SELECT WEEK('2026-04-28', 1);  -- mode=1 周一起始
SELECT WEEK('2026-04-28', 3);  -- mode=3 ISO标准

时间加减运算

DATE_ADD() / ADDDATE() 日期加法

SELECT DATE_ADD('2025-06-11', INTERVAL 7 DAY);   -- 2025-06-18
SELECT DATE_ADD('2025-06-11', INTERVAL 2 MONTH);  -- 2025-08-11
SELECT DATE_ADD('2025-06-11', INTERVAL 1 YEAR);   -- 2026-06-11
SELECT ADDDATE('2025-06-11', 30);                  -- 2025-07-11

DATE_SUB() / SUBDATE() 日期减法

SELECT DATE_SUB('2025-06-11', INTERVAL 7 DAY);   -- 2025-06-04
SELECT DATE_SUB('2025-06-11', INTERVAL 2 MONTH);  -- 2025-04-11
SELECT SUBDATE('2025-06-11', 30);                  -- 2025-05-12

DATEDIFF() 日期差值

SELECT DATEDIFF('2025-06-11', '2025-06-01');  -- 10
SELECT DATEDIFF('2025-06-01', '2025-06-11');  -- -10
SELECT DATEDIFF(NOW(), '2025-01-01');          -- 161

TIMESTAMPDIFF() 时间戳差值

SELECT TIMESTAMPDIFF(DAY, '2025-06-01', '2025-06-11');    -- 10
SELECT TIMESTAMPDIFF(HOUR, '2025-06-11 10:00:00', '2025-06-11 14:30:00');  -- 4
SELECT TIMESTAMPDIFF(MONTH, '2025-01-01', '2025-06-11');  -- 5
SELECT TIMESTAMPDIFF(YEAR, '2020-06-11', '2025-06-11');   -- 5

Part3 – 字符串函数

字符串长度

函数 说明 示例
LENGTH() 返回字节长度 LENGTH('你好') → 6
CHAR_LENGTH() 返回字符长度 CHAR_LENGTH('你好') → 2
BIT_LENGTH() 返回位长度 BIT_LENGTH('Hello') → 40

字符串搜索

-- 查找子字符串位置
SELECT LOCATE('World', 'Hello World');  -- 7
SELECT LOCATE('o', 'Hello World');      -- 5 (第一个'o'的位置)
SELECT LOCATE('o', 'Hello World', 6);   -- 8 (从位置6开始)

-- 找不到时返回0
SELECT LOCATE('xyz', 'Hello World');  -- 0

字符串分隔提取

-- 根据分隔符分割字符串
SELECT SUBSTRING_INDEX('519276654@qq.com', '@', 1);   -- 519276654
SELECT SUBSTRING_INDEX('519276654@qq.com', '@', -1);  -- qq.com

Part6 – SQL 函数

创建函数模板

CREATE FUNCTION 函数名(参数名 类型) RETURNS 返回值类型
BEGIN
    -- 这里写你要执行的SQL逻辑
    RETURN 具体值 OR 一个查询结果;
END

[!note]

  • SQL 自定义函数 = SQL 版的普通函数
  • 有名字、有参数、有返回值
  • BEGIN...END 相当于函数体大括号
  • 可以封装查询逻辑，反复传参调用

示例

-- 定义函数：获取第N高工资
CREATE FUNCTION getNthHighestSalary(N INT) RETURNS INT
BEGIN
    DECLARE M INT;
    SET M = N - 1;
    RETURN (
        SELECT DISTINCT salary
        FROM Employee
        ORDER BY salary DESC
        LIMIT 1 OFFSET M
    );
END

-- 调用函数
SELECT getNthHighestSalary(2);

[!warning]
LIMIT N-1, 1 会报错，需要先声明变量：

  1. DECLARE M INT → 声明变量 M
  2. SET M = N-1 → 把 N-1 存进 M
  3. 然后才能写 LIMIT M, 1

Part7 – 窗口函数

[!tip]
窗口函数 = 不改变原有行数，在每行数据旁边，额外算出排名、序号、前后值、聚合值

普通聚合函数会合并变少行，窗口函数行数不变，只加新列。

GROUP BY vs 窗口函数

特性 GROUP BY 窗口函数
行数变化 分组后行数变少 原始行数不变
数据保留 只保留分组汇总值 保留原始每行数据
计算方式 合并计算 在每行后追加计算结果

窗口函数语法

函数名() OVER (
    PARTITION BY 字段   -- 可选：分组，类似 GROUP BY
    ORDER BY 字段       -- 可选：窗口内排序
    ROWS/RANGE 范围     -- 可选：限定前后几行
)

常用窗口函数

排名函数

-- 行号（连续）
ROW_NUMBER() OVER(ORDER BY score DESC)

-- 跳跃排名（1,1,3,4...）
RANK() OVER(ORDER BY score DESC)

-- 连续排名（1,1,2,3...）
DENSE_RANK() OVER(ORDER BY score DESC)

偏移函数

-- 取当前行上一行数据
LAG(score) OVER(ORDER BY score)

-- 取当前行下一行数据
LEAD(score) OVER(ORDER BY score)

聚合窗口函数

-- 累加求和
SUM(score) OVER(ORDER BY id) AS 累加分数

-- 窗口内平均值
AVG(score) OVER(PARTITION BY class_id)

-- 窗口内最大值/最小值
MAX(score) OVER(PARTITION BY class_id)
MIN(score) OVER(PARTITION BY class_id)

首尾值函数

-- 取窗口第一行值
FIRST_VALUE(score) OVER(ORDER BY score DESC)

-- 取窗口最后一行值
LAST_VALUE(score) OVER(ORDER BY score DESC ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING)

分组函数

-- 把数据平均分成 n 组
NTILE(3) OVER(ORDER BY score)
