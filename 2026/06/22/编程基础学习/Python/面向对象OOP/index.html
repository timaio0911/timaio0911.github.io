Python 面向对象编程

Part1 – OOP 基础

[!info]

  • 核心思想：以「事物/对象」为中心，把现实世界抽象成类和对象，用对象封装数据和行为
  • 关注点：谁来做（Who to do）
  • 特点：高复用、易维护、易扩展、符合人类思维，是大型项目、框架、系统的首选

面向过程 vs 面向对象

对比维度 面向过程 面向对象
核心 函数、步骤 类、对象
思维方式 自顶向下，分步执行 抽象事物，封装数据
复用性 低（复制代码） 高（继承、封装）
维护性 差（改一处动全身） 好（模块化修改）
适合规模 小型程序 中大型项目

[!tip]
面向过程是「一步步做事」，面向对象是「让对象帮你做事」

快速示例

class Calculator:
    def add(self, a, b):
        return a + b

# 创建对象
cal = Calculator()
result = cal.add(1, 2)

Part2 – 类和对象的概念

类（Class）

[!note]
定义：是对象的「模板/蓝图」，抽象描述一类事物共有的属性（特征）和方法（行为）

# 类的定义
class Product:
    """商品类：描述商品的抽象概念"""
    
    # 类属性
    category = "电子产品"
    
    # 实例方法
    def __init__(self, name, price):
        self.name = name      # 实例属性
        self.price = price    # 实例属性
    
    def display(self):
        return f"{self.name}: ¥{self.price}"

对象（Object）

[!note]
定义：是类的「具体实例」，是根据类创建出来的真实个体

# 创建对象（实例化）
phone = Product("iPhone 15", 6999)
laptop = Product("MacBook Pro", 12999)
tablet = Product("iPad Air", 4999)

# 访问属性和方法
print(phone.name)      # iPhone 15
print(laptop.display()) # MacBook Pro: ¥12999

Part3 – 方法详解

构造方法 init

[!tip]

  • __init__ 是 Python 类的构造方法（初始化方法）
  • 创建对象时自动调用，给对象初始化属性
  • self 代表当前创建的对象自己
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

# 创建对象 → 自动调用 __init__
p1 = Person("小明", 20)
print(p1.name)  # 小明
print(p1.age)   # 20

类方法 @classmethod

[!info]

  • 属于类本身，不属于实例对象
  • 操作共享属性时，常使用这个
  • 第一个固定参数：cls（代表当前类）
class Dog:
    species = "哺乳动物"

    @classmethod
    def show_type(cls):
        print("物种：", cls.species)

# 类名调用（推荐）
Dog.show_type()

# 实例对象也能调用
d = Dog()
d.show_type()

工厂方法 —— 快速创建实例

class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    @classmethod
    def from_birth(cls, name, birth_year):
        age = 2026 - birth_year
        return cls(name, age)

# 用类方法工厂创建
p2 = Person.from_birth("李四", 2000)
print(p2.name, p2.age)  # 李四 26

静态方法 @staticmethod

[!note]

  • 属于类，但不绑定 self、也不绑定 cls
  • 就是一个放在类里的普通函数
  • 满足三点就用：逻辑和对象无关、和类本身也无关、只是跟这个类业务相关的工具函数
class MathUtil:
    @staticmethod
    def add(a, b):
        return a + b

print(MathUtil.add(3, 5))

方法对比

类型 装饰器 首参数 归属
实例方法 无 self 实例对象
类方法 @classmethod cls 类
静态方法 @staticmethod 无 类

魔术方法

[!warning]

  • 以双下划线 __ 开头、双下划线结尾的方法
  • 在特定场景自动触发执行
  • 名字是官方固定的，只能重写覆盖，不能自己创造新的

常见魔术方法：

  • __init__：构造方法
  • __str__：字符串表示
  • __del__：析构方法
  • __call__：可调用对象
  • __getitem__：索引访问

Part4 – 动态添加方法

给单个对象添加方法

[!warning]
给单个对象绑实例方法，必须用 types.MethodType 绑定 self

import types

class Person:
    def __init__(self, name):
        self.name = name

def say_hello(self):
    print(f"你好，我是 {self.name}")

p1 = Person("张三")
p1.say = types.MethodType(say_hello, p1)
p1.say()  # 你好，我是 张三

给类添加方法

class Person:
    def __init__(self, name):
        self.name = name

def run(self):
    print(f"{self.name} 在跑步")

# 给类绑定方法
Person.run = run

# 所有实例都能用
p1 = Person("李四")
p1.run()

p2 = Person("王五")
p2.run()

添加类方法/静态方法

class Person:
    pass

# 动态加类方法
Person.cls_func = classmethod(lambda cls: print("类方法执行"))

# 动态加静态方法
Person.sta_func = staticmethod(lambda: print("静态方法执行"))

Person.cls_func()
Person.sta_func()

Part5 – slots 限制

class Person:
    __slots__ = ["name", "age"]

[!warning]
作用：硬性规定实例对象只能有 name、age 这两个属性，不能随便动态加新属性、也不能动态绑定实例方法

三个例外

  1. 给类动态加方法不受 slots 限制
  2. 静态方法、类方法不受影响
  3. 子类如果自己没写 __slots__，会重新拥有 __dict__
class Student(Person):
    pass  # 没写 slots

s = Student()
s.gender = "女"  # 又可以加了

Part6 – 面向对象三大特性

1. 封装（Encapsulation）

[!tip]
核心：把属性和方法打包到类中，隐藏内部细节，只对外暴露接口

私有化方式

类型 命名方式 外部访问 说明
公有 name 可以 默认方式
保护 _name 可以 约定私有，靠程序员自觉
私有 __name 不可以 Python 会改名 _类名__name

@property 装饰器

把方法伪装成属性一样直接用，不用括号

class Person:
    def __init__(self, age):
        self.__age = age

    @property
    def age(self):
        return self.__age

    @age.setter
    def age(self, new_age):
        if 0 < new_age < 150:
            self.__age = new_age

p = Person(20)
print(p.age)   # 20
p.age = 25     # 通过 setter 修改

2. 继承

[!info]
继承：子类自动拥有父类的所有属性和方法，不用重复写代码

# 父类
class Animal:
    def eat(self):
        print("吃东西")

# 子类 继承 Animal
class Dog(Animal):
    def bark(self):
        print("汪汪叫")

# 子类对象 直接能用父类方法
d = Dog()
d.eat()   # 继承来的
d.bark()  # 自己的

方法重写（覆盖）

class Animal:
    def say(self):
        print("动物叫声")

class Cat(Animal):
    def say(self):
        print("喵喵叫")

c = Cat()
c.say()  # 喵喵叫，执行子类自己的

super() 调用父类方法

class Animal:
    def __init__(self, name):
        self.name = name

class Dog(Animal):
    def __init__(self, name, color):
        super().__init__(name)  # 调用父类的 __init__
        self.color = color

d = Dog("旺财", "黄色")
print(d.name, d.color)  # 旺财 黄色

多继承

class A: pass
class B: pass
class C(A, B):  # 同时继承 A、B
    pass

MRO 方法解析顺序

[!note]
MRO（Method Resolution Order）：多继承时，Python 找方法、找属性的先后查找顺序

class A: pass
class B: pass
class C(A, B): pass

print(C.mro())
# C → A → B → object

核心规则：

  1. 子类永远在父类前面
  2. 多个父类，保持继承时的左右顺序
  3. 同一个类只出现一次，不重复

3. 多态

[!tip]
同一个方法，不同子类继承重写后，父类引用指向子类对象，调用方法会自动执行子类自己的版本

条件：

  1. 必须有继承
  2. 子类重写父类同名方法
  3. 用父类变量接收子类对象
class Animal:
    def speak(self):
        pass

class Dog(Animal):
    def speak(self):
        return "汪汪"

class Cat(Animal):
    def speak(self):
        return "喵喵"

# 多态体现
def animal_speak(animal):
    print(animal.speak())

animal_speak(Dog())  # 汪汪
animal_speak(Cat())  # 喵喵
