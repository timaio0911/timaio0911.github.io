Day06 课程知识汇总：大模型评估

课程概述

本课程是"第五阶段 第6天"的内容，主要讲解大模型的评估方法，包括综合知识评估、数学推理评估、代码生成评估等核心评估维度。

目录

  1. 大模型评估概述
  2. 综合知识评估
  3. 数学推理评估
  4. 代码生成评估
  5. 评估指标与方法
  6. 代码实现详解
  7. 关键知识点总结

一、大模型评估概述

1.1 为什么需要评估？

  • 衡量模型能力的高低
  • 比较不同模型的优劣
  • 发现模型的不足之处
  • 指导模型的优化方向

1.2 评估维度

维度 说明
综合知识 多学科知识理解与推理能力
数学推理 数学问题求解能力
代码生成 代码编写与调试能力
语言理解 文本理解与生成能力
安全性 是否产生有害内容

二、综合知识评估

2.1 MMLU

全称：Measuring Massive Multitask Language Understanding

主要目标：测试模型在大规模、多学科任务上的知识理解与推理能力。

覆盖学科：约57个学科，包括数学、历史、物理、化学、法律、医学、计算机科学等。

题型特点：

  • 英文单项选择题
  • 题库规模约15K

评测指标：准确率（Accuracy）

直观理解：

把模型丢进一个"从小学到大学、从文科到理工科"的大杂烩考试里，看它总体学习水平如何。

2.2 C-Eval

全称：Chinese Evaluation Suite

主要目标：测试模型在中文环境下的学科知识+推理能力。

覆盖学科：52个学科，按难度分层：

  • 初中水平
  • 高中水平
  • 大学公共课程
  • 本科专业课程

题型特点：中文单项选择题

直观理解：

“中文版MMLU”，把模型拉去参加一整套中文试卷。

2.3 AGIEval

全称：AGIEval: A Human-Centric Benchmark for Evaluating Foundation Models

主要目标：面向类AGI的综合评估，强调"真实人类标准化考试场景"。

题目来源：

  • 中国高考、研究生入学考试
  • SAT / LSAT等国际标准化考试
  • 律师资格考试、专业资格考试
  • 数学竞赛等

直观理解：

不是"练习册"，而是一堆真正的"正式考试卷"，看模型能不能混进考场"装成人"。

2.4 GaoKao-Bench

全称：GAOKAO-Bench（高考评测集）

主要目标：专门基于中国高考题构建的评测集。

题型特点：

  • 真实高考题：包括客观题（选择题）和主观题（简答/解答题）
  • 涉及语文、数学、英语、文综、理综等科目

直观理解：

直接让模型"参加一次高考"，看看它是学霸、普通考生，还是"复读生级别"。

三、数学推理评估

3.1 GSM8K

全称：Grade School Math 8K

主要目标：测试模型在小学/初中难度的数学应用题上的多步推理能力。

题型特点：

  • 英文文字描述的小故事/应用题
  • 模型需要输出一个最终数值答案
  • 每道题往往需要2-8步推理

数据规模：约8.5K道题

评测指标：答案准确率

直观理解：

“模型能不能像一个细心的小学生一样，认真一步步算清楚日常数学题？”

3.2 MATH

全称：MATH Dataset

主要目标：测试模型在中学到数学竞赛级别的高难度数学问题求解能力。

题型特点：

  • 题目多来自数学竞赛或高难度练习
  • 通常配有完整的LaTeX形式解题过程
  • 模型需要给出最终答案，有时也要求给出推理步骤

数据规模：约12,500道题

直观理解：

“这不是看你会不会算账，而是考你能不能当奥数竞赛选手。”

四、代码生成评估

4.1 HumanEval

来源：OpenAI HumanEval

主要目标：评估模型的代码生成和函数实现能力（主要是Python）。

题型特点：

  • 函数签名（函数名+参数）
  • 自然语言描述（需求说明）
  • 示例（输入/输出样例）

评测方式：

  • 对生成的代码运行隐藏测试用例
  • 检查代码是否能正确执行并输出正确结果

4.2 MBPP

全称：Mostly Basic Python Programming

主要目标：测试模型解决基础Python编程问题的能力。

题型特点：

  • 简单的编程任务
  • 通常只需要几行代码
  • 配有测试用例

五、评估指标与方法

5.1 常用评估指标

指标 说明 适用场景
准确率（Accuracy） 正确答案比例 选择题、分类任务
Pass@k 生成k个代码中至少一个正确的概率 代码生成
BLEU 机器翻译质量 翻译任务
ROUGE 摘要质量 摘要任务
F1 Score 精确率和召回率的调和平均 信息抽取

5.2 评估方法

方法 说明
自动评估 使用指标自动计算分数
人工评估 人工判断模型输出质量
LLM评估 使用强模型（如GPT-4）评估弱模型
竞技场评估 用户投票选择更好的回答（如Chatbot Arena）

5.3 评估注意事项

问题 说明
数据污染 训练数据可能包含测试集
过拟合基准 模型可能专门为评估基准优化
评估偏差 不同评估方法可能有不同偏好
实际应用 基准分数高不代表实际应用效果好

六、代码实现详解

本节介绍如何使用 lm_eval 工具在自定义数据集上对模型进行评估，包括数据集格式、任务配置和评估命令。

6.1 自定义评估数据集格式

评估数据集采用 JSON 格式，每条数据包含 instruction（指令）、input（输入）和 output（期望输出）三个字段。以下为 self_data.json 的示例：

[
  {
    "instruction": "识别并解释给定列表中的两个科学理论：细胞理论和日心说。",
    "input": "",
    "output": "细胞理论是生物科学的一个理论，它认为所有生命体都是由微小的基本单元——细胞所构成......"
  },
  {
    "instruction": "输入三支篮球队的名称并生成一个适当的口号。",
    "input": "输入：俄克拉荷马城雷霆队，芝加哥公牛队，布鲁克林网队。",
    "output": "输出：\"雷霆，公牛和网队：各显神通，角逐群雄!\""
  }
]

说明：

  • instruction：对模型的指令描述，每条数据必填
  • input：附加输入信息，可为空字符串
  • output：期望的模型输出，作为评估的参考答案（Ground Truth）

6.2 自定义评估任务 YAML 配置

使用 lm_eval 评估自定义数据集需要编写 YAML 配置文件（如 custom_task.yaml），定义任务名称、数据来源、提示模板和评估指标：

task: custom_task_eval
dataset_path: json
dataset_kwargs:
  data_files:
    test: self_data.json
test_split: test

output_type: generate_until

# Prompt模板
doc_to_text: |
  {% if input != "" %}
  ###指令
  {{instruction}}
  ###输入
  {{input}}

  {% else %}
  ###指令
  {{instruction}}

  {% endif %}
  ###回答

doc_to_target: "{{output}}"

# 生成参数
generation_kwargs:
  until:
    - "###指令"
    - "</think>"
    - "</s>"
  do_sample: false
  max_gen_toks: 512

# 评估指标
metric_list:
  - metric: rouge
    aggregation: mean
    higher_is_better: true

配置要点：

  • task：自定义任务名称，用于在命令行中引用
  • dataset_path：设为 json 表示使用 JSON 格式数据集
  • dataset_kwargs.data_files：指定数据集文件路径
  • doc_to_text：Prompt 模板，使用 Jinja2 语法，根据是否有 input 字段动态生成提示
  • doc_to_target：期望输出的引用字段
  • generation_kwargs：生成参数，包括停止标记（until）、是否采样（do_sample）和最大生成长度（max_gen_toks）
  • metric_list：评估指标，此处使用 ROUGE 分数（取均值，越高越好）

6.3 使用 lm_eval 进行评估的命令

配置好 YAML 文件后，使用以下命令启动评估：

lm_eval --model hf \
    --model_args pretrained=模型路径 \
    --include_path YAML文件所在目录 \
    --tasks custom_task_eval \
    --batch_size 4 \
    --device cuda:0

参数说明：

参数 说明
--model hf 使用 HuggingFace 模型格式
--model_args pretrained=... 指定本地模型权重路径
--include_path 指定包含自定义 YAML 配置的目录路径
--tasks custom_task_eval 指定要执行的任务名称（对应 YAML 中的 task 字段）
--batch_size 4 推理批大小，根据 GPU 显存调整
--device cuda:0 指定推理使用的 GPU 设备

6.4 评估结果说明

评估完成后，lm_eval 会输出以下关键信息：

  • 指标名称与分数：例如 ROUGE 分数，表示模型生成文本与参考答案之间的相似度
  • 样本数量（N）：参与评估的样本总数
  • 指标均值（mean）：所有样本指标分数的平均值，反映模型整体表现

ROUGE 分数范围为 0~1：

  • 越接近 1，说明模型输出与参考答案越相似
  • 越接近 0，说明模型输出与参考答案差异越大

通过自定义数据集评估，可以针对特定领域或任务场景检验模型的实际能力，补充通用基准评估的不足。

七、关键知识点总结

7.1 综合知识评估

评估集 语言 学科数 特点
MMLU 英文 57 最常用的综合评估
C-Eval 中文 52 中文版MMLU
AGIEval 中英 多 真实考试题
GaoKao-Bench 中文 多 中国高考题

7.2 数学推理评估

评估集 难度 题量 特点
GSM8K 小学/初中 8.5K 多步推理
MATH 竞赛级 12.5K 高难度

7.3 代码生成评估

评估集 语言 特点
HumanEval Python 函数实现
MBPP Python 基础编程

八、总结

本课程系统讲解了：

  1. 综合知识评估：MMLU、C-Eval、AGIEval、GaoKao-Bench
  2. 数学推理评估：GSM8K、MATH
  3. 代码生成评估：HumanEval、MBPP
  4. 评估指标：准确率、Pass@k、BLEU、ROUGE
  5. 评估方法：自动评估、人工评估、LLM评估、竞技场评估

掌握大模型评估方法，是理解和改进模型性能的关键。
