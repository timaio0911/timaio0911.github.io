Day11 课程知识汇总：低代码平台与智能体开发

课程概述

本课程是"第五阶段 第11天"的内容，主要讲解低代码平台（Coze、Dify）的使用，以及如何使用这些平台搭建RAG系统、智能体和工作流应用。

目录

  1. RAG基础知识
  2. 低代码平台介绍
  3. Coze平台使用
  4. Dify平台使用
  5. 实战案例

一、RAG基础知识

1.1 什么是RAG？

**RAG（Retrieval-Augmented Generation，检索增强生成）**是一种结合信息检索与文本生成的技术，旨在提升大语言模型在回答专业问题时的准确性和可靠性。

1.2 为什么需要RAG？

问题 说明
知识时效性 模型知识存在滞后，无法回答最新问题
领域知识缺失 模型可能不了解特定领域知识
无法确定信息来源 无法确定回答的可靠性
幻觉问题 模型可能编造不存在的信息

1.3 RAG执行流程

用户问题
    ↓
[检索前处理] → 查询转换、查询扩充
    ↓
[检索] → 从知识库中检索相关文档
    ↓
[检索后处理] → 重排序、过滤
    ↓
[生成] → LLM基于检索结果生成答案
    ↓
输出答案

1.4 RAG vs 微调

对比项 RAG 微调
知识更新 实时更新 需要重新训练
成本 低 高
幻觉控制 好 较差
适用场景 知识问答 风格调整

二、低代码平台介绍

2.1 什么是低代码平台？

低代码平台是通过可视化界面和少量代码来构建应用的开发平台，大大降低了开发门槛。

2.2 主流低代码平台

平台 特点 适用场景
Coze 字节跳动出品，功能丰富 智能体开发、工作流编排
Dify 开源，支持本地部署 RAG系统、Agent开发
FastGPT 专注知识库问答 企业知识库
RAGFlow 深度文档理解 复杂文档处理

2.3 知识库搭建平台对比

平台 定位 特点
AnythingLLM 桌面AI助手 简单易用，适合个人
CherryStudio 桌面AI助手 图形化界面，快速验证
Dify LLM应用平台 功能强大，支持工作流
FastGPT 知识库问答 专注知识库场景

三、Coze平台使用

3.1 什么是Coze？

Coze是字节跳动推出的AI应用开发平台，支持创建智能体、搭建工作流、发布应用。

3.2 Coze核心功能

功能 说明
智能体 创建具有特定角色的AI助手
工作流 可视化编排复杂的业务流程
插件 调用外部API和工具
知识库 上传文档构建RAG系统
发布 一键发布到多个平台

3.3 Coze智能体创建

步骤1：创建智能体

  • 设定角色和人设
  • 配置提示词

步骤2：添加能力

  • 添加知识库
  • 添加插件
  • 配置工作流

步骤3：测试和发布

  • 测试对话效果
  • 发布到飞书、微信等平台

3.4 Coze工作流

工作流是将多个节点连接起来，形成复杂的业务流程：

开始 → 知识库检索 → LLM生成 → 条件判断 → 输出

常用节点：

  • 开始节点：接收用户输入
  • 知识库节点：检索知识库
  • LLM节点：调用大模型
  • 条件节点：分支判断
  • 代码节点：执行自定义代码
  • 结束节点：输出结果

四、Dify平台使用

4.1 什么是Dify？

Dify是一个开源的LLM应用开发平台，支持本地部署，功能强大。

4.2 Dify核心功能

功能 说明
聊天助手 创建对话式AI应用
文本生成 创建文本生成应用
Agent 创建具有工具调用能力的智能体
工作流 可视化编排复杂流程
知识库 上传文档构建RAG系统

4.3 Dify安装部署

Docker部署：

git clone https://github.com/langgenius/dify.git
cd dify/docker
docker compose up -d

4.4 Dify知识库创建

步骤1：创建知识库

  • 上传文档（PDF、Word、Markdown等）
  • 选择分段方式

步骤2：配置嵌入模型

  • 选择嵌入模型（如text-embedding-ada-002）
  • 配置API Key

步骤3：创建应用

  • 选择应用类型
  • 关联知识库
  • 配置提示词

步骤4：测试和发布

  • 测试问答效果
  • 发布应用

五、实战案例

5.1 智能电商客服

需求：搭建一个能够回答商品问题、处理售后的智能客服。

实现步骤：

  1. 创建知识库：上传商品信息、售后政策
  2. 创建工作流：
    用户输入 → 意图识别 → 知识库检索 → LLM生成 → 输出
  3. 配置提示词：设定客服角色和回答规范
  4. 测试发布：测试对话效果，发布到电商平台

5.2 产品营销海报生成

需求：根据产品信息自动生成营销海报文案。

实现步骤：

  1. 创建智能体：设定营销专家角色
  2. 添加插件：接入图片生成API
  3. 创建工作流：
    产品信息 → 文案生成 → 图片生成 → 输出海报
  4. 测试发布：测试生成效果

5.3 客户投诉分类助手

需求：自动对客户投诉进行分类和优先级排序。

实现步骤：

  1. 创建知识库：上传投诉处理规范
  2. 创建工作流：
    投诉内容 → 情感分析 → 分类判断 → 优先级排序 → 输出
  3. 接入钉钉：将应用集成到钉钉
  4. 测试发布：测试分类准确率

5.4 行业调研报告生成

需求：根据行业关键词自动生成调研报告。

实现步骤：

  1. 创建智能体：设定分析师角色
  2. 添加插件：接入搜索引擎、数据API
  3. 创建工作流：
    行业关键词 → 数据收集 → 分析整理 → 报告生成 → 输出
  4. 测试发布：测试报告质量

六、关键知识点总结

6.1 RAG核心流程

  1. 数据索引：加载 → 分割 → 嵌入 → 索引
  2. 数据查询：检索 → 重排序 → 生成

6.2 低代码平台对比

平台 优势 劣势
Coze 功能丰富，易上手 不支持本地部署
Dify 开源，支持本地部署 需要一定技术基础
FastGPT 专注知识库 功能相对单一

6.3 工作流设计原则

  1. 模块化：将复杂流程拆分为简单节点
  2. 可复用：设计通用节点，方便复用
  3. 可测试：每个节点可独立测试
  4. 可监控：添加日志和监控节点

6.4 智能体设计原则

  1. 明确角色：设定清晰的角色和职责
  2. 丰富提示词：提供详细的指令和示例
  3. 工具调用：接入必要的外部工具
  4. 记忆机制：保持对话上下文

七、平台操作实战

7.1 Cherry Studio 操作流程

Cherry Studio 是一款零基础用户也能快速上手的AI入门工具。

步骤1：下载安装

  1. 访问 Cherry Studio 官网
  2. 选择 Windows 系统版本下载
  3. 傻瓜式安装，无需复杂配置

步骤2：登录账户

  • 使用微信扫码登录
  • 登录后可使用免费和付费模型

步骤3：添加模型

  1. 获取API Key：前往模型官网申请

  2. 配置模型：

    • 模型名称：如 qwen3.6-plus
    • API地址：如 https://dashscope.aliyuncs.com
    • API Key：申请的密钥

  3. 支持的模型类型：

    • 文本对话模型
    • 图像生成模型
    • 视频生成模型
    • 语音合成模型

步骤4：创建知识库

  1. 创建知识库：

    • 点击"知识库"功能
    • 输入知识库名称（如"抖音文案库"）
    • 选择嵌入模型（如智元BGM3）

  2. 上传文档：

    • 支持 PDF、Word、Markdown 等格式
    • 系统自动解析和分段

  3. 高级设置：

    • 文档处理工具：MinerU、OCR
    • 重排序模型：选择免费或自定义模型
    • 分块大小：512-1024字符
    • 重叠大小：50-100字符
    • 匹配阈值：0.5-0.7

步骤5：创建智能助手

  1. 创建助手：

    • 设定助手名称和角色
    • 配置提示词

  2. 关联知识库：

    • 选择已创建的知识库
    • 设置检索模式（意图识别/强制检索）

  3. 测试发布：

    • 测试问答效果
    • 发布到微信、飞书等平台

7.2 Dify 操作流程

Dify 是一个开源的LLM应用开发平台，支持本地部署。

步骤1：Docker部署

# 克隆项目
git clone https://github.com/langgenius/dify.git

# 进入Docker目录
cd dify/docker

# 启动服务
docker compose up -d

步骤2：创建知识库

  1. 上传文档：

    • 支持 PDF、Word、Markdown、TXT 等格式
    • 选择分段方式（自动/手动）

  2. 配置嵌入模型：

    • 选择模型（如 text-embedding-ada-002）
    • 配置 API Key

  3. 设置检索参数：

    • Top K：返回最相似的文档数量
    • 相似度阈值：0.5-0.7

步骤3：创建应用

  1. 选择应用类型：

    • 聊天助手
    • 文本生成
    • Agent
    • 工作流

  2. 配置应用：

    • 关联知识库
    • 设置提示词
    • 选择模型

  3. 测试发布：

    • 测试问答效果
    • 发布应用

7.3 腾讯混元知识库操作

腾讯混元 提供基于微信生态的知识库服务。

操作流程

  1. 下载安装：

    • 支持客户端、小程序、网页端
    • 微信扫码登录

  2. 创建知识库：

    • 创建个人知识库
    • 上传本地文件（如刑法文档）

  3. 使用知识库：

    • 基于知识库提问
    • 支持对话模式和任务模式

7.4 底饭平台操作

底饭 是一个开源大语言模型应用平台。

操作流程

  1. 访问平台：

    • 网页版直接使用
    • 或本地部署

  2. 创建知识库：

    • 导入文本文件
    • 设置分段规则

  3. 配置检索：

    • 选择嵌入模型
    • 设置检索策略
