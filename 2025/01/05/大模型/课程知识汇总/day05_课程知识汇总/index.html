Day05 课程知识汇总：大模型训练介绍

课程概述

本课程是"第五阶段 第5天"的内容，主要讲解大模型训练的三大阶段：预训练、指令微调（SFT）、偏好对齐（RLHF），以及训练基础设施和分布式策略。

目录

  1. 大模型训练的三大阶段
  2. 预训练
  3. 指令微调（SFT）
  4. 偏好对齐（RLHF）
  5. 分布式训练
  6. 微调技术

一、大模型训练的三大阶段

大模型从"随机初始化"到"可用产品"，通常经历预训练 → 指令微调 → 偏好对齐的完整链路。

预训练（通识教育）→ 指令微调（专业培训）→ 偏好对齐（价值观塑造）
阶段 目标 数据量 计算量
预训练 学习语言规律和世界知识 万亿级token 非常大
指令微调 学会听从指令 十万至百万级样本 相对较小
偏好对齐 符合人类价值观 数万至数十万条对比数据 中等

二、预训练

2.1 核心目标

让模型掌握语言的通用规律、世界知识和基础推理能力。

2.2 技术原理

自监督学习（Self-supervised Learning）：无需人工标注，从文本自身结构构造监督信号。

Next Token Prediction：给定前文，预测下一个token的概率分布。

$$
P(x_t | x_{<t}; \theta)
$$

最大化序列的联合概率：

$$
\prod_{t=1}^{T} P(x_t | x_{<t})
$$

等价于最小化交叉熵损失：

$$
\mathcal{L} = -\sum_{t} \log P(x_t | x_{<t})
$$

2.3 数据特点

维度 说明 示例
规模 数千亿至数万亿token The Pile、Common Crawl
多样性 覆盖多领域、多语言 维基百科、学术论文、GitHub代码
质量筛选 去重、过滤低质内容 MinHash去重、分类器过滤

2.4 算力需求

模型 参数量 训练数据 硬件需求
GPT-3 175B 3000亿token 数千张V100，数月
LLaMA 3 70B 15万亿token 16000张H100

三、指令微调（SFT）

3.1 核心目标

将"语言模型"转变为"指令跟随模型"，使其能理解并执行人类指令。

3.2 数据格式

{
  "messages": [
    {"role": "system", "content": "你是一个有帮助的AI助手。"},
    {"role": "user", "content": "请用Python写一个快速排序算法。"},
    {"role": "assistant", "content": "以下是快速排序的实现..."}
  ]
}

3.3 关键设计

设计要点 说明
多样性覆盖 问答、摘要、翻译、代码、数学推理等
对话模板标准化 不同模型使用特定的chat template
数据质量 > 数量 数十万条高质量数据比数百万条低质数据更有效

3.4 与预训练对比

维度 预训练 SFT
数据量 万亿级token 十万至百万级样本
计算量 很大 相对较小
目标 通用语言建模 指令跟随与对话能力
参数更新 全参数随机初始化→收敛 基于预训练权重继续训练

四、偏好对齐（RLHF）

4.1 核心问题

SFT后的模型可能产生有害、偏见或不符合人类偏好的输出。

4.2 RLHF流程

收集偏好数据 → 训练奖励模型 → RL优化策略模型
(人工排序对比)   (学习人类偏好)   (PPO算法)

4.3 步骤详解

步骤1：构建偏好数据集

  • 对同一提示生成多个回答（A、B）
  • 人工标注者选择更优回答：A≻B
  • 通常需要数万至数十万条对比数据

步骤2：训练奖励模型（Reward Model）

  • 输入：(prompt, response)
  • 输出：标量奖励值
  • 损失函数：最大化被偏好回答与被拒绝回答的奖励差距

步骤3：强化学习优化（PPO）

  • 目标：最大化奖励，同时约束与参考模型的KL散度
  • 策略梯度更新，需要大量采样和在线生成

4.4 替代方案

方案 说明
RLAIF 使用强模型（如GPT-4）替代人类进行偏好标注
DPO 跳过显式奖励模型，直接用偏好数据优化策略

五、分布式训练

5.1 为什么需要分布式训练？

大模型参数规模带来的挑战：

挑战 说明
内存墙 175B参数FP16存储需350GB，远超单卡显存
计算墙 单次前向传播需数千亿次计算
通信墙 多卡之间需要高效同步梯度与参数

5.2 三种并行策略

策略 切分对象 适用场景 通信开销
数据并行（DP） 数据批次 模型可放入单卡 梯度同步
模型并行（MP） 层内参数 单层参数量过大 激活值传输
流水线并行（PP） 模型层 模型层数多 气泡开销

3D并行：现代大模型训练通常组合三种策略。

5.3 关键优化技术

技术 说明
ZeRO 优化器状态、梯度、参数分片
混合精度训练 FP16/BF16前向计算 + FP32主权重更新
梯度检查点 时间换空间，减少显存占用30%-70%
FlashAttention IO感知的注意力计算，提速2-4倍

5.4 主流训练框架

框架 特点 适用场景
Megatron-LM NVIDIA开发，3D并行优化极致 超大规模预训练
DeepSpeed 微软开源，ZeRO优化 全阶段训练
Llama-Factory 一站式微调框架 社区微调首选

六、微调技术

6.1 全参数微调（Full Fine-tuning）

适用场景：

  • 领域差异极大（如通用模型→医疗专业模型）
  • 需要深度改变模型行为
  • 拥有充足算力

特点：

  • 更新所有参数
  • 需要大量显存
  • 效果最好但成本最高

6.2 参数高效微调（PEFT）

核心思想：只更新少量参数，冻结大部分预训练权重。

方法 原理 参数量
LoRA 低秩矩阵分解 0.1%-1%
QLoRA 量化+LoRA 更少
Prefix Tuning 在输入前添加可训练前缀 很少
Adapter 在层间插入小型网络 1%-5%

6.3 LoRA原理

核心思想：将权重更新分解为低秩矩阵。

$$
W’ = W + \Delta W = W + BA
$$

其中：

  • W：原始预训练权重（冻结）
  • B：低秩矩阵（d×r）
  • A：低秩矩阵（r×d）
  • r：秩，远小于d

优势：

  • 参数量大幅减少（0.1%-1%）
  • 显存需求低
  • 训练速度快
  • 可以保存多个LoRA适配器

七、关键知识点总结

7.1 训三大阶段

阶段 目标 数据 方法
预训练 学习语言规律 万亿token Next Token Prediction
指令微调 学会听指令 十万样本 有监督学习
偏好对齐 符合人类价值观 对比数据 RLHF/DPO

7.2 分布式训练

  • 数据并行：切分数据批次
  • 模型并行：切分层内参数
  • 流水线并行：切分模型层

7.3 微调技术

  • 全参数微调：更新所有参数，效果最好
  • LoRA：低秩矩阵分解，参数高效
  • QLoRA：量化+LoRA，更省显存

八、LLaMA Factory详解

8.1 什么是LLaMA Factory？

LLaMA Factory是一个一站式大模型微调框架，支持100+种模型的微调，包括：

  • LLaMA、Qwen、ChatGLM、Mistral等
  • 支持SFT、RLHF、DPO等多种训练方式
  • 提供Web UI界面，无需编写代码

8.2 安装LLaMA Factory

# 克隆仓库
git clone https://github.com/hiyouga/LLaMA-Factory.git
cd LLaMA-Factory

# 安装依赖
pip install -e .

# 或者直接pip安装
pip install llama-factory

功能 说明 命令

8.4 LLaMA Factory配置文件

SFT配置示例（sft.yaml）：

### model
model_name_or_path: Qwen/Qwen2.5-7B-Instruct  # 基座模型

### method
stage: sft                    # 训练阶段：sft/dpo/ppo
finetuning_type: lora         # 微调类型：full/lora/qlora
lora_target: all              # LoRA目标层
lora_rank: 8                  # LoRA秩

### dataset
dataset: alpaca_zh             # 数据集名称
template: qwen                 # 对话模板
cutoff_len: 1024              # 最大长度

### output
output_dir: saves/qwen2.5-7b/lora/sft  # 输出目录
logging_steps: 10             # 日志间隔
save_steps: 500               # 保存间隔
plot_loss: true               # 绘制损失曲线

### train
per_device_train_batch_size: 2    # 批次大小
gradient_accumulation_steps: 4    # 梯度累积
learning_rate: 1.0e-4             # 学习率
num_train_epochs: 3.0             # 训练轮数
lr_scheduler_type: cosine         # 学习率调度
bf16: true                        # 使用BF16

### eval
val_size: 0.1                     # 验证集比例
per_device_eval_batch_size: 2     # 验证批次大小
eval_strategy: steps              # 验证策略
eval_steps: 500                   # 验证间隔

9.2 加载SFT后的模型（01_transformer中加载sft后的模型使用.py）

from transformers import AutoTokenizer, AutoModel

# 加载微调后的模型
tokenizer = AutoTokenizer.from_pretrained(
    "saves/qwen2.5-7b/lora/sft/merged"
)
model = AutoModel.from_pretrained(
    "saves/qwen2.5-7b/lora/sft/merged"
)

# 准备输入
messages = [
    {"role": "user", "content": "你是谁？你会什么？"},
]

# 应用对话模板
inputs = tokenizer.apply_chat_template(
    messages,
    add_generation_prompt=True,
    tokenize=True,
    return_dict=True,
    return_tensors="pt",
).to(model.device)

# 生成回答
outputs = model.generate(**inputs, max_new_tokens=40)
print(tokenizer.decode(outputs[0][inputs["input_ids"].shape[-1]:]))

9.3 调用LLaMA Factory API（02_通过openai向llamafactory部署的api发送请求.py）

9.4 使用Transformers Pipeline

from transformers import pipeline

# 创建pipeline
pipe = pipeline(
    "text-generation",
    model="saves/qwen2.5-7b/lora/sft/merged"
)

# 生成文本
messages = [
    {"role": "user", "content": "你是谁？你会什么？"},

  4. 分布式训练：数据并行、模型并行、流水线并行
  5. 微调技术：全参数微调、LoRA、QLoRA
  6. LLaMA Factory：一站式微调框架，支持SFT/DPO/PPO，提供Web UI和API部署

掌握这些训练技术，是进行大模型开发的核心能力。
