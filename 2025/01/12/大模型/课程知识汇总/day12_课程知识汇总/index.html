Day12 课程知识汇总：大模型RAG技术深入

课程概述

本课程是"第五阶段 第12天"的内容，主要深入讲解RAG（检索增强生成）技术，包括RAG的原理、架构演进、优化策略以及实际应用。

目录

  1. 大模型的局限性
  2. RAG技术详解
  3. RAG架构演进
  4. RAG优化策略
  5. RAG应用场景

一、大模型的局限性

1.1 知识时效性

LLM需要花费大量时间和资源进行预训练，因此会存在知识滞后问题，无法回答训练时间点之后的问题。

1.2 领域知识缺失

LLM掌握了大量通用知识，但在细分专业领域可能训练数据不足，导致无法准确回答相关问题。

1.3 无法确定信息来源

LLM在生成回答时，无法确定其回答内容的依据，也就无法确定回答的可靠性。

1.4 幻觉问题

LLM在生成回答时，可能会"一本正经的胡说八道"，这被称为"幻觉"问题。

幻觉产生的原因：

  • 训练知识存在偏差
  • 过度泛化
  • 缺乏深层理解
  • 缺乏特定领域知识

二、RAG技术详解

2.1 什么是RAG？

**RAG（Retrieval-Augmented Generation，检索增强生成）**是将传统的生成式LLM和实时信息检索技术相结合，为LLM补充来自外部的相关数据和上下文，帮助LLM生成更加准确可靠的内容。

RAG就相当于在LLM回答时给它一本参考书。

2.2 RAG核心流程

数据索引阶段

步骤 说明
加载（Loading） 解析不同格式的文档，转换为可处理的数据
分割（Splitting） 将大内容分割成小的知识块
嵌入（Embedding） 将知识块转换为向量
索引（Indexing） 将向量存储到向量数据库

数据查询阶段

步骤 说明
检索前处理 查询转换、查询扩充、检索路由
检索（Retrieval） 从知识库中检索相关知识块
检索后处理 重排序、过滤
生成（Generation） LLM基于检索结果生成答案

2.3 RAG涉及的大模型

模型类型 作用 示例
Embedding模型 将文本转换为向量 text-embedding-ada-002、bge-m3
Rerank模型 对检索结果重排序 bge-reranker、cohere-rerank
LLM 生成最终答案 GPT-4、Qwen、DeepSeek

2.4 Reranker的使用场景

场景 是否使用Reranker
追求高精度和高相关性 ✅ 推荐使用
响应时间要求高 ❌ 不推荐使用
成本敏感 ❌ 不推荐使用

三、RAG架构演进

3.1 Naive RAG（朴素RAG）

特点：遵循传统的基础流程：索引、检索、生成。

问题：

  • 检索召回精度低
  • 召回率不足
  • 生成质量不稳定

3.2 Advanced RAG（进阶RAG）

改进：

  • 检索前处理：查询转换、查询扩充
  • 检索后处理：重排序、过滤
  • 更好的分块策略

3.3 Modular RAG（模块化RAG）

特点：

  • 将RAG流程拆分为独立模块
  • 模块可自由组合
  • 支持多种检索策略

3.4 Graph RAG（图RAG）

特点：

  • 使用知识图谱替代向量数据库
  • 支持复杂关系推理
  • 适合需要多跳推理的场景

四、RAG优化策略

4.1 分块策略优化

策略 说明 适用场景
固定大小分块 按字符数分割 通用场景
语义分块 按语义边界分割 文档结构清晰
递归分块 多层级分割 复杂文档
文档结构分块 按标题、段落分割 结构化文档

4.2 检索优化

优化方法 说明
混合检索 结合向量检索和关键词检索
查询扩展 生成多个相关查询
重排序 使用Reranker对结果重排序
过滤 过滤低质量结果

4.3 生成优化

优化方法 说明
提示词优化 设计更好的提示词模板
上下文压缩 压缩检索结果，减少token使用
多轮对话 保持对话上下文
引用标注 标注信息来源

4.4 向量数据库选择

数据库 特点 适用场景
ChromaDB 轻量级，易于使用 快速原型开发
Pinecone 云服务，性能好 生产环境
Weaviate 开源，功能丰富 需要高级功能
Milvus 开源，高性能 大规模数据
Qdrant 开源，Rust实现 高性能需求

五、RAG应用场景

5.1 企业知识库问答

需求：构建企业内部知识库，员工可以快速查询公司政策、流程等信息。

实现：

  1. 上传公司文档（员工手册、政策文件等）
  2. 构建向量索引
  3. 创建问答应用

5.2 技术文档智能助手

需求：为技术文档创建智能助手，帮助开发者快速查找API文档、使用示例等。

实现：

  1. 加载技术文档（API文档、README等）
  2. 构建向量索引
  3. 创建问答应用

5.3 法律/医疗领域专业咨询

需求：构建法律或医疗领域的专业咨询系统。

实现：

  1. 上传专业文档（法律条文、医学指南等）
  2. 构建向量索引
  3. 配置专业提示词
  4. 创建问答应用

5.4 个性化内容推荐

需求：根据用户兴趣推荐相关内容。

实现：

  1. 构建内容库
  2. 分析用户兴趣
  3. 检索相关内容
  4. 生成推荐理由

六、关键知识点总结

6.1 RAG核心概念

  • 检索增强生成：结合检索和生成的技术
  • 向量检索：使用向量相似度进行语义检索
  • 知识块：将文档分割成小块用于检索
  • 重排序：对检索结果进行二次排序

6.2 RAG流程

数据索引：加载 → 分割 → 嵌入 → 索引
数据查询：检索前处理 → 检索 → 检索后处理 → 生成

6.3 RAG优化要点

方面 优化方法
分块 语义分块、递归分块
检索 混合检索、查询扩展
重排序 使用Reranker
生成 提示词优化、上下文压缩

6.4 RAG vs 微调

对比项 RAG 微调
知识更新 实时 需重新训练
成本 低 高
幻觉控制 好 较差
适用场景 知识问答 风格调整

7.3 文本清洗

代码文件：03_文本清洗.py

import json
import re

def clean_content(documents: list):
    """清洗文档内容"""
    cleaned_docs = []
    for document in documents:
        # 获取文本
        text = document.page_content

        # 使用正则表达式进行清洗
        # 1. 将连续的2个及以上的换行符，替换为2个换行符
        text = re.sub("\n{2,}", "\n\n", text)
        text = text.strip()

        # 2. 修改元数据中的内容，将所有非支持的数值类型替换为JSON格式字符串
        allowed_type = (str, int, float, bool)
        for key, value in document.metadata.items():
            if not isinstance(value, allowed_type):
                try:
                    document.metadata[key] = json.dumps(value, ensure_ascii=False)
                except:
                    document.metadata[key] = str(value)

        document.page_content = text
        cleaned_docs.append(document)

    return cleaned_docs

    CharacterTextSplitter,
    RecursiveCharacterTextSplitter,

    SentenceTransformersTokenTextSplitter

    separator="。",
    chunk_size=512,
    chunk_overlap=100,
)

# 2. 多字符递归切分

    separators=["。", "、", "，"],
    chunk_size=512,
    chunk_overlap=100,
)

# 3. Token切分

    chunk_size=512,
    chunk_overlap=100,
)

# 4. SentenceTransformers切分（推荐）
text_splitter = SentenceTransformersTokenTextSplitter(
    model_name="bge-base-zh-v1.5",  # 本地模型路径
    chunk_size=512,
    chunk_overlap=100
)

# 执行切分
text_split = text_splitter.split_documents(documents)

7.7 相似度检索

代码文件：07_相似度检索.py

from langchain_chroma import Chroma
from langchain_huggingface import HuggingFaceEmbeddings

embedding_model = HuggingFaceEmbeddings(
    model_name="bge-base-zh-v1.5"
)

# 加载存储的向量
vector_stores = Chroma(
    persist_directory="db/sample_vector_db",
    embedding_function=embedding_model
)

# 检索方式1：余弦相似度检索
query = "请输入你的问题："
sim_docs = vector_stores.similarity_search(query, k=3)

# 检索方式2：最大边际相关性检索（推荐）
sim_docs = vector_stores.max_marginal_relevance_search(query, k=3)

# 使用检索器
retriever = vector_stores.as_retriever(
    search_type="mmr",  # 或 "similarity"
    search_kwargs={"k": 3}
)
sim_docs = retriever.invoke(query)

for doc in sim_docs:
    print("*" * 20)
    print(doc.page_content)

7.8 RAG增强生成

代码文件：08_增强生成.py 和 10_增强生成.py

from langchain_chroma import Chroma
from langchain_huggingface import HuggingFaceEmbeddings
from langchain_community.llms.ollama import Ollama
from langchain_core.prompts import PromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables.passthrough import RunnablePassthrough

# 1. 加载模型和向量数据库
llm = Ollama(model="qwen3.5:0.8b")

embedding_model = HuggingFaceEmbeddings(
    model_name="bge-base-zh-v1.5"
)

vector_stores = Chroma(
    persist_directory="db/sample_vector_db",
    embedding_function=embedding_model
)

# 2. 创建检索器
retriever = vector_stores.as_retriever(
    search_type="mmr",
    search_kwargs={"k": 3}
)

# 3. 定义提示词模板
prompt = PromptTemplate(
    input_variables=["context", "query"],
    template="""
    你是一个专业的中文问答助手，擅长基于检索到的内容回答问题。
    请仅根据以下的背景资料回答，如果无法找到答案，请直接回答不知道。

    背景资料：{context}
    问题：{query}

    回答：
    """
)

# 4. 格式化检索结果
def format_str(docs):
    return "\n".join([doc.page_content for doc in docs])

# 5. 构建RAG链
rag_chain = (
    {"context": retriever | format_str, "query": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

# 6. 使用RAG链
while True:
    query = input("请输入你的问题：")
    if query == "退出":
        break

    print("助手的回答：")
    result = rag_chain.stream(query)
    for chunk in result:
        print(chunk, end="")
    print()

7.9 完整RAG项目示例

代码文件：09_构建索引.py 和 10_增强生成.py

构建中医问答系统的完整流程：

# 1. 加载多个文档
from langchain_community.document_loaders import UnstructuredWordDocumentLoader
from langchain_text_splitters import SentenceTransformersTokenTextSplitter

file_paths = [
    r"knowledge_base\中医临床诊疗术语 第1部分：疾病.docx",
    r"knowledge_base\中医临床诊疗术语 第2部分：证候.docx",
    r"knowledge_base\中医临床诊疗术语 第3部分：治法.docx"
]

all_documents = []
for path in file_paths:
    loader = UnstructuredWordDocumentLoader(path, mode="single", strategy="hi_res")
    docs = loader.load()
    all_documents.extend(docs)

# 2. 文本切分
text_splitter = SentenceTransformersTokenTextSplitter(
    model_name="bge-base-zh-v1.5",
    chunk_size=64,
    chunk_overlap=10
)
text_split = text_splitter.split_documents(all_documents)

# 3. 创建向量数据库
from langchain_chroma import Chroma
from langchain_huggingface import HuggingFaceEmbeddings

embedding_model = HuggingFaceEmbeddings(model_name="bge-base-zh-v1.5")
vectorstore = Chroma.from_documents(
    text_split,
    embedding_model,
    persist_directory="db/chinese_medicine_vector_db"
)

# 4. 创建RAG链
from langchain_community.llms.ollama import Ollama
from langchain_core.prompts import PromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables.passthrough import RunnablePassthrough

retriever = vectorstore.as_retriever(
    search_type="similarity",
    search_kwargs={"k": 3}
)

prompt = PromptTemplate(
    input_variables=["context", "query"],
    template="""
    你是一个专业的中医问答助手，擅长基于检索到内容的回答问题。
    请根据以下的背景资料回答，如果无法找到答案，请直接回答不知道。

    背景资料：{context}
    问题：{query}

    回答：
    """
)

def format_str(docs):
    return "\n".join([doc.page_content for doc in docs])

llm = Ollama(model="qwen3.5:4b")

rag_chain = (
    {"context": retriever | format_str, "query": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

# 5. 使用问答系统
while True:
    query = input("请输入你的问题：")
    if query == "退出":
        break

    print("助手的回答：")
    result = rag_chain.stream(query)
    for chunk in result:
        print(chunk, end="")
    print()
