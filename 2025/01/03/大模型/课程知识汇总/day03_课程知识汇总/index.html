一、大模型基础概念

1.1 什么是大模型？

**大模型（Large Language Model, LLM）**是一种参数量非常大的深度学习模型，具有以下特点：

特点 说明
参数量大 以B（10亿）为单位，如0.8B、4B、32B等
基于Transformer 使用Transformer架构，通常只用解码器部分
自监督学习 通过预测下一个token进行学习
通用能力强 可完成聊天、写作、推理、搜索等多种任务

1.2 大模型的工作原理

大模型的核心是预测下一个token：

输入: "今天天气"
模型预测: "真" (概率0.8)
输入: "今天天气真"
模型预测: "好" (概率0.9)
输入: "今天天气真好"
模型预测: "！" (概率0.7)

关键点：

  • 大模型不是百科全书，而是"成语接龙"高手
  • 模型通过大量数据学习语言规律
  • 生成过程本质上是分类任务（从词表中选择下一个词）

1.3 大模型的训练需求

需求 说明
数据量 PB级（千万亿级别）数据
GPU资源 大量显存，如32B模型需64GB+显存
训练时间 数天甚至数周
电力成本 高性能GPU功耗大

二、大模型参数量与存储

2.1 参数量单位

单位 含义 示例
B (Billion) 10亿 0.8B = 8亿参数
T (Trillion) 万亿 1T = 1万亿参数

2.2 参数存储格式

BF16（Brain Floating Point 16）：

  • 16位浮点型
  • 每个参数占用2字节
  • 现代大模型普遍采用

2.3 存储空间计算

公式：

存储空间 = 参数量 × 2字节

示例：

模型 参数量 存储空间
0.8B 8亿 1.6GB
4B 40亿 8GB
32B 320亿 64GB

计算过程：

0.8B模型 = 852,959,200个参数
每个参数 = 2字节
总空间 = 852,959,200 × 2 = 1,705,918,400字节
       = 1,705,918,400 / 1024 / 1024 / 1024 ≈ 1.58GB

2.4 训练显存需求

训练时除了模型参数，还需要存储：

  • 网络结构
  • 优化器状态
  • 梯度信息

经验公式：

训练显存 ≈ 模型参数存储 × 3~4倍

例如：0.8B模型训练需要约5GB显存

三、大模型核心能力

3.1 自然语言理解与生成

能力 说明
语义分析 理解文本含义
意图识别 识别用户目的
上下文理解 理解前后文关系
文本生成 根据指令生成内容

3.2 多模态处理

多模态是指处理多种类型数据的能力：

模态转换 示例
文本→图片 根据描述生成图片
文本→视频 根据脚本生成视频
图片→文本 图片描述生成
语音→文本 语音识别

3.3 上下文能力

上下文窗口决定模型一次能处理多少文本：

时期 上下文长度 说明
早期 ~1K tokens 只能处理短文本
现代 128K~1M tokens 可处理整本书

影响因素：

  • 自注意力机制的QK点乘运算消耗大量显存
  • 上下文越长，显存需求越大

3.4 其他核心能力

能力 说明
强泛化能力 可完成多种任务
少样本迁移 通过少量示例学习新任务
可扩展性 通过微调适应专业领域

四、提示词工程基础

4.1 什么是提示词工程？

**提示词工程（Prompt Engineering）**是通过设计优质提示词来引导大模型输出的技术。

4.2 提示词四要素

要素 说明 示例
Role（角色） 设定模型身份 “你是一名资深分析师”
Context（背景） 提供上下文信息 “我需要向董事会汇报”
Instruction（指令） 明确任务要求 “请列出报告大纲”
Constraint（约束） 限制输出格式 “字数500字内”

4.3 CRISPE框架

维度 定义 示例
Capacity 能力/角色 你是资深敏捷开发专家
Receipt 接受对象 读者是初级产品经理
Instruction 核心指令 将需求转化为User Story
Statement 细节描述 包含角色、功能点、验收标准
Personality 语气风格 专业、严谨、模块化
Experiment 示例 提供示例格式

4.4 高阶技巧

Few-Shot（少样本学习）：

例子1：极智科技获得红杉2亿融资。 -> {公司: 极智科技, 金额: 2亿}
例子2：云深处完成亿元战略融资。 -> {公司: 云深处, 金额: 1亿}
任务：大模型指挥官公司获种子轮500万美元。 ->

CoT（思维链）：

"请一步步思考：首先核对金额上限，其次核对消费类别。"

五、API调用大模型

5.1 常见大模型API

平台 模型 特点
OpenAI GPT-4, GPT-5 国际领先，效果好
阿里云 通义千问 国内领先，性价比高
DeepSeek DeepSeek-V4 国产之光，效果优秀
Ollama 各种开源模型 本地部署，免费使用

5.2 OpenAI API调用

from openai import OpenAI

client = OpenAI(
    api_key="your-api-key",
    base_url="https://api.openai.com/v1"
)

response = client.responses.create(
    model="gpt-4",
    input="介绍一下你自己！"
)

print(response.output_text)

5.3 通义千问API调用

from openai import OpenAI

client = OpenAI(
    api_key="your-api-key",
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1"
)

response = client.chat.completions.create(
    model="qwen-turbo",
    messages=[
        {"role": "user", "content": "介绍一下你自己！"}
    ]
)

print(response.choices[0].message.content)

5.4 Ollama本地调用

from llama_index.llms.ollama import Ollama

llm = Ollama(
    model="qwen3.5:0.8b",
    base_url="http://localhost:11434",
    request_timeout=120,
    context_window=1024
)

response = llm.complete("介绍一下你自己！")
print(response)

六、Ollama本地部署

6.1 什么是Ollama？

Ollama是一个开源的本地大模型运行框架，可以让你在自己的电脑上运行各种开源大模型，无需联网，完全免费。

核心特点：

  • 支持多种开源模型（Qwen、Llama、Mistral等）
  • 本地运行，无需联网
  • 简单易用，一行命令即可运行
  • 支持API调用，兼容OpenAI格式

6.2 Ollama安装

Windows安装：

  1. 设置环境变量（可选，自定义模型存储路径）：
setx OLLAMA_MODELS d:\ollama\models /M

  1. 下载并安装Ollama：

    • 访问 https://ollama.ai 下载安装包
    • 运行 OllamaSetup.exe /dir=d:\ollama

  2. 验证安装：

ollama -v

6.3 Ollama常用命令

命令 说明 示例
ollama pull 下载模型 ollama pull qwen3.5:0.8b
ollama run 运行模型 ollama run qwen3.5:0.8b
ollama list 查看已下载模型 ollama list
ollama rm 删除模型 ollama rm qwen3.5:0.8b
ollama create 创建自定义模型 ollama create mymodel -f Modelfile
ollama show 查看模型信息 ollama show qwen3.5:0.8b

6.4 常用模型

模型 大小 说明 下载命令
qwen3.5:0.8b 0.5GB 通义千问，轻量级 ollama pull qwen3.5:0.8b
qwen3.5:4b 2.5GB 通义千问，平衡版 ollama pull qwen3.5:4b
llama3.1:8b 4.7GB Meta开源模型 ollama pull llama3.1:8b
mistral:7b 4.1GB Mistral AI模型 ollama pull mistral:7b
deepseek-coder:6.7b 3.8GB 代码专用模型 ollama pull deepseek-coder:6.7b

6.5 Ollama API调用

Ollama启动后会自动提供API服务，默认地址为 http://localhost:11434。

基本调用：

from openai import OpenAI

client = OpenAI(
    api_key="ollama",  # 随意填写
    base_url="http://localhost:11434/v1"
)

response = client.chat.completions.create(
    model="qwen3.5:0.8b",
    messages=[
        {"role": "user", "content": "介绍一下你自己！"}
    ]
)

print(response.choices[0].message.content)

流式输出：

from openai import OpenAI

client = OpenAI(
    api_key="ollama",
    base_url="http://localhost:11434/v1"
)

response = client.chat.completions.create(
    model="qwen3.5:0.8b",
    messages=[
        {"role": "user", "content": "介绍一下你自己！"}
    ],
    stream=True  # 启用流式输出
)

for chunk in response:
    result = chunk.choices[0].delta.content
    if result:
        print(result, end="", flush=True)
print()

6.6 Ollama自定义模型（Modelfile）

Modelfile是Ollama的模型定义文件，可以基于现有模型创建自定义模型。

Modelfile格式：

FROM qwen3.5:0.8b
SYSTEM "你是一名耐心细致的 AI 助教，名字叫张三，擅长用中文讲解机器学习和大模型相关知识。"
PARAMETER temperature 0.7

常用参数：

参数 说明 默认值
FROM 基础模型 必填
SYSTEM 系统提示词 空
PARAMETER temperature 温度（0-2） 0.7
PARAMETER top_p 核采样 0.9
PARAMETER top_k Top-K采样 40
PARAMETER num_ctx 上下文长度 2048
PARAMETER num_gpu GPU层数 自动

创建自定义模型：

# 1. 创建Modelfile文件
# 2. 运行创建命令
ollama create mymodel -f Modelfile

# 3. 运行自定义模型
ollama run mymodel

调用自定义模型：

from openai import OpenAI

client = OpenAI(
    api_key="ollama",
    base_url="http://localhost:11434/v1"
)

response = client.chat.completions.create(
    model="mymodel",  # 使用自定义模型名称
    messages=[
        {"role": "user", "content": "你好，请介绍一下你自己！"}
    ],
    stream=True
)

for chunk in response:
    result = chunk.choices[0].delta.content
    if result:
        print(result, end="", flush=True)
print()

6.7 Ollama vs 云端API对比

对比项 Ollama 云端API
成本 免费 按量付费
隐私 数据本地处理 数据上传云端
网络 无需联网 需要联网
性能 依赖本地硬件 云端高性能
模型选择 开源模型 商业模型
易用性 需要安装 直接调用

6.8 Ollama硬件要求

模型大小 最低内存 推荐内存 最低显存
0.5B-1B 4GB 8GB 2GB
3B-4B 8GB 16GB 4GB
7B-8B 16GB 32GB 8GB
13B-14B 32GB 64GB 16GB
70B+ 64GB 128GB 48GB+
