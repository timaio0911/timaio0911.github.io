Day07 课程知识汇总：大模型推理部署

课程概述

本课程是"第五阶段 第7天"的内容，主要讲解大模型的推理部署，包括主流推理框架介绍、vLLM使用、Docker容器化部署等。

目录

  1. 主流推理框架
  2. vLLM详解
  3. Docker部署
  4. 量化技术
  5. 部署最佳实践

一、主流推理框架

1.1 框架对比

框架 核心优势 适用场景 部署难度
vLLM 高吞吐、PagedAttention 生产环境、Python开发者 ⭐
TensorRT-LLM NVIDIA硬件极致性能 追求极致速度 ⭐⭐⭐
SGLang 复杂Prompt缓存、结构化输出 Agent开发、多轮对话 ⭐⭐
llama.cpp CPU/Mac推理、低资源 本地运行、边缘设备 ⭐
TGI 生产级容器、监控完善 Kubernetes集群部署 ⭐⭐
LMDeploy 4-bit量化性能极强 追求速度且显存有限 ⭐⭐
MLC LLM 跨平台（手机/Web） 移动端应用 ⭐⭐⭐

1.2 各框架详解

vLLM：

  • PagedAttention技术，高效管理KV Cache
  • 高吞吐量，适合高并发场景
  • 原生支持OpenAI兼容API

TensorRT-LLM：

  • NVIDIA官方推出，针对NVIDIA GPU极致优化
  • In-flight Batching高效动态批处理
  • 需要编译模型为TensorRT引擎

SGLang：

  • RadixAttention自动复用KV Cache
  • 结构化输出支持极好（JSON、Regex）
  • 特别适合Agent推理流程

llama.cpp：

  • GGUF格式，体积小，显存需求低
  • Apple Silicon优化，Mac首选
  • 纯C/C++实现，无依赖

TGI：

  • Hugging Face官方生产级推理容器
  • Rust核心，安全性与性能兼顾
  • 支持分布式追踪、Prometheus监控

LMDeploy：

  • TurboMind自研高效推理引擎
  • 4bit量化下性能通常优于vLLM
  • 包含量化、推理、服务化全套工具

二、vLLM详解

2.1 什么是vLLM？

vLLM是一个高吞吐量、内存友好的大模型推理与服务框架，核心是用PagedAttention管理KV Cache。

和直接用transformers推理相比，vLLM的特点是：更省显存、更高吞吐、更适合高并发服务。

2.2 核心特性

特性 说明
高吞吐/高并发 持续批处理，请求随到随混批
PagedAttention KV Cache分页管理，减少碎片
分布式并行 支持Tensor Parallel、Pipeline Parallel
OpenAI兼容API 直接支持/v1/chat/completions等接口
广泛模型支持 支持LLaMA、Qwen、Mixtral等主流模型
多硬件平台 支持NVIDIA CUDA、AMD ROCm、CPU等

2.3 安装方式

pip安装：

pip install -U vllm

Docker安装（推荐）：

docker pull vllm/vllm-openai:latest

2.4 使用示例

启动服务：

python -m vllm.entrypoints.openai.api_server \
    --model Qwen/Qwen2.5-7B-Instruct \
    --host 0.0.0.0 \
    --port 8000

调用API：

from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:8000/v1",
    api_key="not-needed"
)

response = client.chat.completions.create(
    model="Qwen/Qwen2.5-7B-Instruct",
    messages=[
        {"role": "user", "content": "你好！"}
    ]
)

print(response.choices[0].message.content)

三、Docker部署

3.1 什么是Docker？

Docker是一个开源的应用容器引擎，可以将应用及其依赖打包到一个可移植的容器中。

3.2 Docker核心概念

概念 说明
镜像（Image） 只读模板，包含运行应用所需的一切
容器（Container） 镜像的运行实例
仓库（Registry） 存储镜像的服务（如Docker Hub）
Dockerfile 定义镜像构建过程的文本文件

3.3 常用Docker命令

# 拉取镜像
docker pull vllm/vllm-openai:latest

# 运行容器
docker run -d --gpus all \
    -p 8000:8000 \
    -v ~/.cache/huggingface:/root/.cache/huggingface \
    vllm/vllm-openai:latest \
    --model Qwen/Qwen2.5-7B-Instruct

# 查看运行中的容器
docker ps

# 停止容器
docker stop <container_id>

# 查看容器日志
docker logs <container_id>

3.4 Docker Compose

version: '3.8'
services:
  vllm:
    image: vllm/vllm-openai:latest
    ports:
      - "8000:8000"
    volumes:
      - ~/.cache/huggingface:/root/.cache/huggingface
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities: [gpu]
    command: --model Qwen/Qwen2.5-7B-Instruct

四、量化技术

4.1 什么是量化？

量化是将模型参数从高精度（如FP32、FP16）转换为低精度（如INT8、INT4）的技术。

4.2 量化类型

类型 说明 精度损失 速度提升
FP16 16位浮点 很小 1.5-2x
BF16 Brain Float 16 很小 1.5-2x
INT8 8位整数 小 2-3x
INT4 4位整数 中等 3-4x

4.3 常用量化方法

方法 说明
GPTQ 训练后量化，需要校准数据
AWQ 激活感知量化，效果好
GGUF llama.cpp使用的量化格式
BitsAndBytes 动态量化，易于使用

4.4 量化示例

from transformers import AutoModelForCausalLM, BitsAndBytesConfig

# 4bit量化配置
quantization_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_compute_dtype=torch.float16,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_use_double_quant=True
)

# 加载量化模型
model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen2.5-7B-Instruct",
    quantization_config=quantization_config,
    device_map="auto"
)

五、部署最佳实践

5.1 硬件选择

模型大小 推荐GPU 显存需求
0.5B-1B RTX 3060/4060 4-8GB
3B-7B RTX 3090/4090 16-24GB
13B-14B A100 40GB 32-40GB
70B+ A100 80GB × 多卡 80GB+

5.2 性能优化

优化策略 说明
量化 使用INT8/INT4减少显存占用
批处理 合理设置batch_size提高吞吐
KV Cache 使用PagedAttention优化显存
异步推理 使用异步API提高并发能力

5.3 监控与告警

监控项 说明
GPU利用率 显卡使用情况
显存占用 内存使用情况
请求延迟 响应时间
吞吐量 每秒处理请求数
错误率 请求失败比例

六、关键知识点总结

6.1 推理框架选择

场景 推荐框架
生产环境、高并发 vLLM
极致性能、NVIDIA GPU TensorRT-LLM
Agent开发、多轮对话 SGLang
本地运行、低资源 llama.cpp
Kubernetes部署 TGI
4bit量化 LMDeploy

6.2 vLLM核心优势

  • PagedAttention高效显存管理
  • 持续批处理高吞吐
  • OpenAI兼容API
  • 广泛模型支持

6.3 Docker部署优势

  • 环境一致性
  • 快速部署
  • 易于扩展
  • 资源隔离

6.4 量化技术

  • FP16/BF16：精度损失小，速度提升1.5-2x
  • INT8：精度损失小，速度提升2-3x
  • INT4：精度损失中等，速度提升3-4x

七、总结

本课程系统讲解了：

  1. 主流推理框架：vLLM、TensorRT-LLM、SGLang、llama.cpp等
  2. vLLM使用：PagedAttention、OpenAI兼容API、高并发支持
  3. Docker部署：容器化部署、Docker Compose
  4. 量化技术：FP16、INT8、INT4、GPTQ、AWQ
  5. 部署最佳实践：硬件选择、性能优化、监控告警

掌握大模型推理部署技术，是将模型投入实际应用的关键。
