Day09 课程知识汇总：大模型结构及训练优化

课程概述

本课程是"第五阶段 第9天"的内容，主要讲解大模型的结构（Transformer、MoE）以及训练优化技术。

目录

  1. Transformer架构详解
  2. MoE架构
  3. 训练优化技术
  4. 长上下文优化

一、Transformer架构详解

1.1 Transformer的优势

优势 说明
全序列并行计算 所有位置可以一起算，训练效率极高
强大的全局依赖建模 任意两个位置可以互相"看见"
结构规则、易扩展 易于加深、加宽、加长
通用性强 一套架构适配大量任务

1.2 Transformer为什么能成为"基座模型"？

  1. Attention能捕获长距离依赖
  2. 训练非常高效（完全并行）
  3. 极易扩展（层数、宽度、上下文）
  4. 单一架构能覆盖90% NLP任务

Transformer = 通用可扩展计算骨架 + 大规模数据训练 → 变成所有AI能力的"底座"

1.3 Transformer的缺点与局限

问题 说明
O(n²)注意力 计算和显存开销随序列长度平方增长
上下文长度有限 超长上下文会导致推理变慢、显存占用暴涨
缺乏结构先验 需要更多数据和算力学习规律
推理仍是顺序生成 一个token接一个token地出

1.4 Transformer核心组件

组件 作用
Self-Attention 捕捉序列内部依赖关系
Multi-Head Attention 并行学习不同类型的依赖
Feed Forward Network 非线性变换
残差连接 缓解梯度消失
Layer Normalization 稳定训练

二、MoE架构

2.1 什么是MoE？

**Mixture of Experts（MoE，专家混合）**是一种用于扩展大模型规模、提升计算效率的神经网络架构。

核心思想：

不再让所有参数参与每次前向计算，而是把大模型拆成多个"专家子模型"，每次只激活其中一小部分来处理当前输入。

这样就形成了一个**"参数很多，但每次只用一点"的稀疏大模型**。

2.2 MoE核心组件

组件 作用
专家（Experts） 多个专门处理特定问题的子网络
门控网络（Gating Network） 决定哪个专家处理当前输入
路由器（Router） 根据输入选择最合适的专家

2.3 MoE工作流程

训练阶段：

  1. 专家培训：每个专家在特定数据上训练
  2. 训练门控网络：学习如何为输入选择最合适的专家
  3. 联合训练：专家和门控网络共同训练

推理阶段：

  1. 输入通过门控网络
  2. 门控网络选择Top-K个专家
  3. 只有被选中的专家参与计算
  4. 综合专家输出得到最终结果

2.4 MoE优势

优势 说明
参数量大但计算量小 总参数量很大，但每次只用一小部分
专家专门化 不同专家可以专门处理不同类型的任务
可扩展性 可以通过增加专家数量来扩展模型

2.5 MoE代表模型

模型 专家数 参数量 特点
Mixtral 8x7B 8 46.7B 每次激活2个专家
DeepSeek-V2 160 236B 每次激活6个专家
GPT-4 据传16 1.8T 未官方确认

三、训练优化技术

3.1 混合精度训练

原理：前向传播使用FP16/BF16，反向传播使用FP32。

优势：

  • 显存占用减少约50%
  • 训练速度提升1.5-2倍
  • 精度损失很小

实现：

from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()

with autocast():
    output = model(input)
    loss = criterion(output, target)

scaler.scale(loss).backward()
scaler.step(optimizer)
scaler.update()

3.2 梯度累积

原理：将多个小batch的梯度累积后再更新参数。

适用场景：

  • 显存不足，无法使用大batch
  • 需要模拟大batch训练效果

实现：

accumulation_steps = 4

for i, (input, target) in enumerate(dataloader):
    output = model(input)
    loss = criterion(output, target)
    loss = loss / accumulation_steps
    loss.backward()

    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()

3.3 梯度检查点

原理：用时间换空间，只保存部分激活值，需要时重新计算。

优势：显存占用减少30%-70%

劣势：训练速度降低约20%-30%

实现：

from torch.utils.checkpoint import checkpoint

class Model(nn.Module):
    def forward(self, x):
        x = checkpoint(self.layer1, x)
        x = checkpoint(self.layer2, x)
        return x

3.4 FlashAttention

原理：IO感知的注意力计算，减少HBM访问。

优势：

  • 训练速度提升2-4倍
  • 显存占用减少
  • 支持更长的序列

实现：

from flash_attn import flash_attn_func

# 使用FlashAttention
output = flash_attn_func(q, k, v, causal=True)

四、长上下文优化

4.1 长上下文的挑战

挑战 说明
O(n²)复杂度 注意力计算开销随序列长度平方增长
显存占用 KV Cache占用大量显存
位置编码外推 超出训练长度时性能下降

4.2 RoPE位置编码

全称：Rotary Position Embedding

原理：将位置信息编码为旋转矩阵。

优势：

  • 支持相对位置编码
  • 具有良好的外推性
  • 计算高效

4.3 ALiBi位置编码

全称：Attention with Linear Biases

原理：在注意力分数上添加线性偏置。

优势：

  • 无需额外参数
  • 良好的外推性
  • 实现简单

4.4 长上下文优化方法

方法 原理 效果
Sliding Window Attention 只关注局部窗口 减少计算量
Sparse Attention 稀疏注意力模式 减少计算量
Ring Attention 分布式序列并行 支持超长序列
KV Cache压缩 压缩缓存的KV 减少显存占用

五、关键知识点总结

5.1 Transformer架构

  • 完全基于注意力机制
  • 全序列并行计算
  • 易于扩展（加深、加宽、加长）
  • O(n²)复杂度是主要瓶颈

5.2 MoE架构

  • 参数很多，但每次只用一部分
  • 门控网络选择专家
  • Mixtral、DeepSeek-V2等代表模型

5.3 训练优化技术

技术 效果
混合精度训练 显存减少50%，速度提升2x
梯度累积 模拟大batch训练
梯度检查点 显存减少30%-70%
FlashAttention 速度提升2-4x

5.4 长上下文优化

  • RoPE/ALiBi位置编码
  • Sliding Window Attention
  • Sparse Attention
  • Ring Attention

六、总结

本课程系统讲解了：

  1. Transformer架构：优势、局限、核心组件
  2. MoE架构：专家混合、门控网络、代表模型
  3. 训练优化：混合精度、梯度累积、梯度检查点、FlashAttention
  4. 长上下文优化：RoPE、ALiBi、稀疏注意力

掌握这些模型结构和优化技术，是进行大模型开发和训练的基础。
