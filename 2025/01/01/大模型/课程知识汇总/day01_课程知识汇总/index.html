Day01 课程知识汇总：编码器-解码器架构与注意力机制

课程概述

本课程是"第五阶段 第1天"的内容，主要讲解编码器-解码器架构及其在序列到序列任务中的应用，并引入注意力机制来提升模型性能。课程以中英文翻译为实战项目，从零搭建一个完整的翻译系统。

一、核心概念

1.1 什么是序列到序列（Seq2Seq）任务？

序列到序列任务是指将一个序列转换为另一个序列的任务，典型应用包括：

任务 输入 输出 示例
机器翻译 源语言句子 目标语言句子 “我爱北京” → “I love Beijing”
语音识别 音频信号 文本 语音 → 文字
文本摘要 长文本 短摘要 文章 → 摘要
图像描述 图像特征 文本描述 图片 → “一只猫在睡觉”

1.2 为什么需要编码器-解码器？

直接翻译（如用一个RNN从输入直接映射到输出）存在以下问题：

  • 输入和输出长度可能不同
  • 无法捕捉输入序列的完整语义信息
  • 难以处理长距离依赖关系

编码器-解码器架构通过将任务分解为两步来解决这些问题：

  1. 编码器：理解输入序列，提取特征
  2. 解码器：根据编码器的输出生成目标序列

二、编码器-解码器架构详解

2.1 整体架构

输入序列: x₁, x₂, x₃, ..., xₙ
         ↓
    [编码器 Encoder]
         ↓
上下文向量: C = f(h₁, h₂, ..., hₙ)
         ↓
    [解码器 Decoder]
         ↓
输出序列: y₁ → y₂ → y₃ → ... → yₘ

2.2 编码器（Encoder）

作用：将输入序列压缩成一个固定长度的上下文向量

关键组件：

  • Embedding层：将单词转换为向量表示
  • GRU层：处理序列信息，提取特征

工作流程：

# 伪代码
输入: "欢迎 来 北京"
    ↓ Embedding
向量: [v1, v2, v3]
    ↓ GRU（双向）
隐藏状态: [h1, h2, h3]
    ↓ 取最后时刻的隐藏状态
上下文向量: C = [h3_forward, h3_backward]  # 拼接前向和反向

代码实现（models_encoder_decoder.py）：

class Encoder(torch.nn.Module):
    def __init__(self, vocab_size, embedding_dim=128, hidden_size=128, padding_idx=0):
        super(Encoder, self).__init__()
        self.embedding = torch.nn.Embedding(vocab_size, embedding_dim, padding_idx=padding_idx)
        self.gru = torch.nn.GRU(
            input_size=embedding_dim,
            hidden_size=hidden_size,
            num_layers=2,           # 2层GRU
            bidirectional=True,     # 双向GRU
            batch_first=True
        )

    def forward(self, x):
        x = self.embedding(x)    # B L D -> [批次大小, 序列长度, 嵌入维度]
        output, h_t = self.gru(x)

        # 双向GRU的隐藏状态：
        # h_t[-2]: 前向传播最后一层的隐藏状态
        # h_t[-1]: 反向传播最后一层的隐藏状态
        last_hidden_forward = h_t[-2]
        last_hidden_backward = h_t[-1]

        # 拼接前向和反向的隐藏状态，形成上下文向量
        context_vector = torch.cat([last_hidden_forward, last_hidden_backward], dim=-1)
        return context_vector

2.3 解码器（Decoder）

作用：根据编码器的上下文向量，逐步生成目标序列

关键组件：

  • Embedding层：将目标单词转换为向量
  • GRU层：基于上下文向量生成序列
  • 全连接层：将GRU输出映射到词汇表大小，进行分类

工作流程：

# 伪代码
输入: 上下文向量C + 上一个单词（训练时用真实标签，推理时用预测结果）
    ↓ Embedding
向量: [v]
    ↓ GRU（初始隐藏状态为C）
输出: output, h_t
    ↓ 全连接层
预测: 每个单词的概率分布
    ↓ argmax
下一个单词

代码实现（models_encoder_decoder.py）：

class Decoder(torch.nn.Module):
    def __init__(self, vocab_size, embedding_dim=128, hidden_size=256, padding_idx=0):
        super(Decoder, self).__init__()
        self.embedding = torch.nn.Embedding(vocab_size, embedding_dim, padding_idx=padding_idx)
        self.gru = torch.nn.GRU(
            input_size=embedding_dim,
            hidden_size=hidden_size,
            num_layers=1,           # 单层GRU
            batch_first=True
        )
        self.fc = torch.nn.Linear(hidden_size, vocab_size)  # 全连接层进行分类

    def forward(self, x, h_0):  # h_0 是编码器提取的向量，作为GRU的初始隐藏状态
        x = self.embedding(x)
        output, h_t = self.gru(x, h_0)  # 使用编码器的输出作为初始隐藏状态
        output = self.fc(output)         # 全连接层映射到词汇表大小
        return output, h_t

2.4 训练与推理的区别

阶段 输入来源 技术 说明
训练时 真实标签 Teacher Forcing 使用真实目标词作为下一步输入，加速收敛
推理时 模型预测 自回归生成 使用模型自己的预测作为下一步输入

Teacher Forcing示例：

训练翻译 "欢迎来北京" → "welcome to Beijing"

时间步1: 输入<SOS>，预测"welcome"，但实际输入"welcome"（真实标签）
时间步2: 输入"welcome"，预测"to"，但实际输入"to"（真实标签）
时间步3: 输入"to"，预测"Beijing"，但实际输入"Beijing"（真实标签）
时间步4: 输入"Beijing"，预测<EOS>，结束

三、注意力机制详解

3.1 为什么需要注意力机制？

传统编码器-解码器的问题：

  1. 信息瓶颈：整个输入序列被压缩成固定长度的向量
  2. 长序列丢失：序列越长，早期信息丢失越严重
  3. 无法聚焦：解码器无法选择性地关注输入的不同部分

生活类比：

  • 传统方式：把一本书的内容全部压缩成一句话，然后根据这句话来回答问题
  • 注意力机制：回答问题时，可以翻阅书的不同页面，找到最相关的部分

3.2 注意力机制的核心思想

为输入的不同部分分配不同的"注意力"权重

在翻译每个词时，模型会关注输入序列中最相关的部分：

翻译 "我爱北京天安门" → "I love Beijing Tiananmen"

生成"I"时：主要关注"我"（权重0.7）
生成"love"时：主要关注"爱"（权重0.8）
生成"Beijing"时：主要关注"北京"（权重0.7）
生成"Tiananmen"时：主要关注"天安门"（权重0.8）

3.3 QKV机制

注意力机制通过计算**查询（Query）、键（Key）、值（Value）**的加权平均值来工作：

符号 名称 来源 含义
Q (Query) 查询向量 解码器当前隐藏状态 “我现在想要什么信息”
K (Key) 键向量 编码器每个时间步输出 “每个输入词包含什么信息”
V (Value) 值向量 编码器每个时间步输出 “每个输入词的内容信息”

生活类比：

  • 在图书馆找关于"特征矩阵"的内容
  • Q = 你的问题"特征矩阵在哪里？"
  • K = 书的目录（索引）
  • V = 书的具体内容

3.4 注意力计算公式

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

计算步骤：

  1. 计算匹配得分：Q和K通过点积计算相似度

    scores = torch.matmul(Q, K.transpose(-2, -1)) / torch.sqrt(torch.tensor(d_k))

  2. 归一化权重：得分经过softmax得到注意力权重

    attention_weights = F.softmax(scores, dim=-1)

  3. 加权求和：将V向量加权平均得到输出

    output = torch.matmul(attention_weights, V)

3.5 带注意力机制的解码器

代码实现（models_encoder_decoder_with_attention.py）：

class Decoder(torch.nn.Module):
    def __init__(self, vocab_size, embedding_dim=128, hidden_size=256, padding_idx=0):
        super(Decoder, self).__init__()
        self.embedding = torch.nn.Embedding(vocab_size, embedding_dim, padding_idx=padding_idx)
        self.gru = torch.nn.GRU(
            input_size=embedding_dim,
            hidden_size=hidden_size,
            num_layers=1,
            batch_first=True
        )
        # 全连接层输入维度变为 hidden_size*2，因为要拼接GRU输出和注意力输出
        self.fc = torch.nn.Linear(hidden_size * 2, vocab_size)

    def forward(self, x, h_0, encoder_outputs):  # 新增参数：编码器的输出
        x = self.embedding(x)
        output, h_t = self.gru(x, h_0)

        # 计算注意力
        q = h_t.transpose(0, 1)       # 解码器当前隐藏状态作为Query
        k = v = encoder_outputs        # 编码器的输出同时作为Key和Value

        attention = F.scaled_dot_product_attention(q, k, v)  # 缩放点积注意力
        output = torch.cat([output, attention], dim=-1)      # 拼接GRU输出和注意力输出

        output = self.fc(output)  # 全连接层
        return output, h_t

关键变化：

  1. forward方法新增参数encoder_outputs
  2. 使用F.scaled_dot_product_attention计算注意力
  3. 将GRU输出和注意力输出拼接后送入全连接层
  4. 全连接层输入维度变为hidden_size*2

4.2 分词器（tokenizer.py）

核心功能：

  1. 分词：将文本分割成单词列表
  2. 编码：将单词转换为数字下标
  3. 解码：将数字下标转换回单词
  4. 词表管理：构建和加载词表

特殊标记：

unk_token = "<unk>"  # 未登录词（Unknown）
pad_token = "<pad>"  # 填充（Padding）
sos_token = "<sos>"  # 句子开始（Start of Sentence）
eos_token = "<eos>"  # 句子结束（End of Sentence）

编码过程（encode方法）：

def encode(self, text, seq_len, add_sos_eos=False):
    word_list = self.tokenize(text)  # 分词

    if add_sos_eos:
        # 添加SOS和EOS标记，并补齐或截断到指定长度
        if len(word_list) == seq_len - 2:
            word_list = [self.sos_token] + word_list + [self.eos_token]
        elif len(word_list) < seq_len - 2:
            word_list = [self.sos_token] + word_list + [self.eos_token] + [self.pad_token] * (seq_len - len(word_list) - 2)
        else:
            word_list = [self.sos_token] + word_list[:(seq_len - 2)] + [self.eos_token]
    else:
        # 只补齐或截断，不添加SOS/EOS
        if len(word_list) >= seq_len:
            word_list = word_list[:seq_len]
        elif len(word_list) < seq_len:
            word_list = word_list + [self.pad_token] * (seq_len - len(word_list))

    # 将单词转换为下标
    return [self.word2index.get(word, self.unk_token_id) for word in word_list]

中文分词 vs 英文分词：

class ChineseTokenizer(BaseTokenizer):
    @staticmethod
    def tokenize(text):
        return jieba.lcut(text)  # 使用结巴分词

class EnglishTokenizer(BaseTokenizer):
    @staticmethod
    def tokenize(text):
        return word_tokenize(text)  # 使用NLTK分词

六、训练与预测流程

6.1 训练流程（02_编码器解码器模型训练.py）

# 1. 初始化
zh_tokenizer = ChineseTokenizer.load_vocab("data/vocab/zh_vocab.txt")
en_tokenizer = EnglishTokenizer.load_vocab("data/vocab/en_vocab.txt")

encoder = Encoder(vocab_size=zh_tokenizer.vocab_size, embedding_dim=128, padding_idx=zh_tokenizer.pad_token_id).to(device)
decoder = Decoder(vocab_size=en_tokenizer.vocab_size, embedding_dim=128, padding_idx=en_tokenizer.pad_token_id).to(device)

# 2. 损失函数和优化器
loss_func = torch.nn.CrossEntropyLoss(ignore_index=en_tokenizer.pad_token_id)  # 忽略padding的损失
optimizer = torch.optim.Adam(params=chain(encoder.parameters(), decoder.parameters()), lr=1e-3)

# 3. 训练循环
for epoch in range(EPOCH):
    for batch, (zh_batch, en_batch) in enumerate(dataloader):
        # 编码
        context_vector = encoder(zh_batch)

        # 解码（Teacher Forcing）
        decoder_input = en_batch[:, 0:1]  # 第一个输入是<SOS>
        decoder_hidden = context_vector.unsqueeze(0)

        decoder_outputs = []
        for t in range(1, en_batch.shape[1]):
            decoder_output, decoder_hidden = decoder(decoder_input, decoder_hidden)
            decoder_outputs.append(decoder_output)
            decoder_input = en_batch[:, t:t+1]  # 使用真实标签作为下一步输入

        # 计算损失
        decoder_outputs = torch.cat(decoder_outputs, dim=1)
        decoder_outputs = decoder_outputs.reshape(-1, decoder_outputs.shape[-1])
        decoder_targets = en_batch[:, 1:].reshape(-1)

        loss = loss_func(decoder_outputs, decoder_targets)

        # 反向传播
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

    # 保存最佳模型
    if ave_loss < best_loss:
        best_loss = ave_loss
        torch.save(encoder.state_dict(), "models/encoder_best.pth")
        torch.save(decoder.state_dict(), "models/decoder_best.pth")

6.2 预测流程（03_编码器解码器模型进行预测.py）

# 1. 加载模型和词表
encoder.load_state_dict(torch.load("models/encoder_best.pth"))
decoder.load_state_dict(torch.load("models/decoder_best.pth"))

# 2. 预测循环
while True:
    zh_text = input("请输入中文：")
    if zh_text == "退出":
        break

    # 编码
    input_indexs = zh_tokenizer.encode(zh_text, seq_len=20, add_sos_eos=False)
    input_tensor = torch.tensor(input_indexs, dtype=torch.long).to(device)
    input_tensor = input_tensor.unsqueeze(0)  # 添加批次维度

    with torch.no_grad():
        context_vector = encoder(input_tensor)

        # 解码（自回归生成）
        decoder_hidden = context_vector.unsqueeze(0)
        decoder_input = torch.full(
            (batch_size, 1),
            fill_value=en_tokenizer.sos_token_id,  # 从<SOS>开始
            dtype=torch.long,
            device=device
        )

        pre_dict_word_ids = []
        for i in range(20):  # 最多预测20个单词
            decoder_output, decoder_hidden = decoder(decoder_input, decoder_hidden)
            predict_index = torch.argmax(decoder_output, dim=-1)  # 选择概率最大的单词

            if predict_index.item() == en_tokenizer.eos_token_id:  # 遇到<EOS>结束
                break

            pre_dict_word_ids.append(predict_index.item())
            decoder_input = predict_index  # 将预测结果作为下一步输入

        # 解码为文本
        words = en_tokenizer.decode(pre_dict_word_ids)
        print(words)

for t in range(1, en_batch.shape[1]):

七、关键知识点总结

7.1 编码器-解码器架构

组件 作用 关键点
Embedding 将单词转换为向量 每个单词对应一个可学习的向量
GRU 处理序列信息 比RNN更好地处理长距离依赖
双向GRU 捕捉前后文信息 前向和反向的隐藏状态拼接
上下文向量 连接编码器和解码器 编码器的最终隐藏状态

7.2 注意力机制

概念 含义
Q (Query) 解码器当前想要什么信息
K (Key) 编码器每个位置包含什么信息
V (Value) 编码器每个位置的具体内容
注意力权重 Q和K的相似度，经过softmax归一化

计算公式：
$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

7.3 训练技巧

  1. Teacher Forcing：训练时使用真实标签作为下一步输入，加速收敛
  2. 梯度裁剪：防止梯度爆炸
  3. 模型保存策略：只保存损失下降时的模型
  4. 忽略Padding：损失函数中忽略padding token的损失

7.4 常见问题与解决方案

问题 原因 解决方案
长序列翻译质量差 信息瓶颈 引入注意力机制
过拟合 数据量不足 增加数据、使用正则化
训练速度慢 批次大小不当 调整batch_size
内存溢出 数据集太大 减小batch_size

7.5 两种模型对比

特性 基础版 带注意力版
编码器 相同 相同
解码器 只使用上下文向量 使用上下文向量 + 注意力
全连接层输入维度 hidden_size hidden_size * 2
长序列处理能力 较差 较好
训练速度 较快 较慢

A.1 双向GRU的隐藏状态处理

# 双向GRU的隐藏状态形状：[num_layers * 2, batch_size, hidden_size]
# h_t[-2]: 前向传播最后一层的隐藏状态
# h_t[-1]: 反向传播最后一层的隐藏状态

last_hidden_forward = h_t[-2]    # 前向
last_hidden_backward = h_t[-1]   # 反向

# 拼接得到上下文向量
context_vector = torch.cat([last_hidden_forward, last_hidden_backward], dim=-1)
# 形状：[batch_size, hidden_size * 2]

A.2 注意力计算

import torch.nn.functional as F

# Q: 解码器当前隐藏状态 [batch_size, 1, hidden_size]
# K: 编码器输出 [batch_size, seq_len, hidden_size]
# V: 编码器输出 [batch_size, seq_len, hidden_size]

q = h_t.transpose(0, 1)
k = v = encoder_outputs

# 缩放点积注意力
attention = F.scaled_dot_product_attention(q, k, v)

# 拼接GRU输出和注意力输出
output = torch.cat([output, attention], dim=-1)

A.3 损失函数配置

# 忽略padding token的损失
loss_func = torch.nn.CrossEntropyLoss(ignore_index=en_tokenizer.pad_token_id)

# 优化器同时更新编码器和解码器的参数
optimizer = torch.optim.Adam(
